# 路线图

本文档概述了 **NautilusTrader** 的关键优先级和即将到来的目标，
描绘了其作为高性能算法交易前沿平台的发展路径。

鉴于项目的动态性质，优先级可能会演变以跟上快速发展的开发周期。
有关实时更新和详细的任务跟踪，请参考 [NautilusTrader 看板](https://github.com/orgs/nautechsystems/projects/3)。

**注意**：错误修复和路线图优先级优先于功能请求，以确保稳定性
和进展。但是，改进和新功能的拉取请求（PR）始终受欢迎。
有关更多详细信息，请参见 [CONTRIBUTING_CN.md](/CONTRIBUTING_CN.md)。

## 愿景

将 NautilusTrader 建立为量化算法交易的标准平台，结合
性能、可靠性、可用性和全面的文档，为交易者和开发者提供支持。

## 优先级

1. **将核心移植到 Rust**

   **目标**：利用 Rust 的性能和安全特性来提高可靠性、性能和可扩展性。
   - 用 Rust 重写性能关键组件（替换现有的 Cython 模块）。
   - 使用 PyO3 确保 Rust 和 Python 层之间的互操作性。
   - 在整个转换过程中基准测试性能改进。

2. **改进文档和教程**

    **目标**：降低新用户的学习曲线，并为开发者提供清晰、全面的指南：
   - 通过添加缺失的部分来填补用户和开发者文档的空白。
   - 添加更多教程和示例。

3. **改进代码人体工程学**

    **目标**：简化用户和贡献者的开发体验：
   - 增强类型注释和对 Python 导入解析的支持。
   - 标准化命名约定并优化 API 以提高直观性。
   - 简化配置和设置过程以最小化摩擦。
   - 重构模块和命名空间以提高可读性和可维护性。

## 其他增强功能

在我们推进顶级优先级的同时，我们还计划专注于以下增强功能：

- 扩展适配器集成以支持交易场所和数据提供商。
- 通过附加功能增强回测引擎。
- 通过附加功能增强订单簿执行动态，包括用户订单交互、持久订单簿更改和扩展的微观结构模拟。

## 开源范围

NautilusTrader 开源项目专门构建以赋能个人和
小团队量化交易者，使策略研究和实时交易在单节点上高效可靠。通过明确定义什么是范围 *内* 和 *外*，
我们设定了明确的期望，集中社区努力，并支持可持续的开源生态系统。

### 范围内

- 高性能单节点回测，准确模拟实时交易条件。
- 在单节点基础设施上进行实时交易，用于简化的研究到生产工作流程。
- 社区贡献的附加交易场所和数据提供商集成：
  - 应首先通过 GitHub 或 Discord 提出和讨论。
  - 必须与 NautilusTrader 的专业、性能聚焦和高可靠性理念保持一致。
  - 必须严格遵循现有的基于 Rust 的适配器实现模式。

### 范围外

- UI 仪表板或可视化器：重点严格保持在核心交易引擎上。
- 分布式或大规模并行回测编排。
  虽然外部编排的工作流程在技术上是兼容的，但内置的分布式运行器超出了项目当前的范围。
- 集成的超参数优化或内置的 AI/ML 工具。用户可以集成自己的优化框架来满足他们的需求。
- 未明确列为范围内的其他外部集成。

## 长期承诺

NautilusTrader 是一个开放核心项目。所有核心交易引擎
功能首先在公共仓库中实现，我们致力于
持续扩大功能集并改进文档，以便社区
可以依赖一个现代、高性能、经过实战考验的平台。

用户的反馈和贡献直接影响路线图；随着
现实世界需求的发展，我们将稳步提高使用
开源代码库可以实现的目标上限。

## NautilusTrader v2.0 及以后

- **实现稳定状态**：虽然 NautilusTrader 已经在生产环境中成功使用，但 v2.0 代表了建立稳定 API 的重要里程碑。
- **重点领域**：v2.0 倡议将优先考虑 API 一致性、长期可维护性，并满足实时交易环境的严格标准。
- **正式弃用**：v2.0 将引入正式弃用，使采用更改和新功能变得更容易，同时为开发者保持清晰度。
- **Python API 承诺**：尽管将核心转换为 Rust，NautilusTrader 将继续提供面向用户的 Python API。

## 规划未来

这个路线图建立在 NautilusTrader 的坚实基础之上，推动持续改进，同时
为算法交易者和开发者扩展其可能性和能力。
