# 示例：使用消息总线进行自通信

实际演示在策略内使用 NautilusTrader 消息总线进行自通信的方法。
该示例实现了一个"第10根K线通知系统"，其中策略：

1. 创建自定义事件（使用 Python 的数据类）来表示第10根K线的出现。
2. 当第10根K线到达时，将此事件发布到消息总线。
3. 在同一策略内订阅并处理这些事件。

**关键学习点：**

- 使用消息总线创建自定义事件。
- 实现发布/订阅模式进行自通信。
- 使用事件进行基于条件的通知。
- 通过消息总线事件处理状态变化。

此模式为在交易策略内处理条件通知
和状态变化提供了清晰的事件驱动方法。

**注意：**
虽然此示例在单个策略内显示了发布者和订阅者角色，但在实践中这些角色
可以分布 - 任何组件都可以是发布者，任何其他组件都可以是事件订阅者。
